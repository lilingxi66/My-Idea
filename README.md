# Labelme 定制版说明

本项目基于上游 `labelme 3.16.2` 定制，主要面向大批量 polygon 标注、标签统一改名、降低误操作和提升边界精修效率。

主要修改文件：
- `app.py`
- `shape.py`
- `widgets/canvas.py`

## 与原始 labelme 3.16.2 的功能对比

| 功能 | 原始 3.16.2 | 当前定制版 |
| --- | --- | --- |
| `Polygon Labels` 原地编辑 | 不支持 | 支持 |
| 图像位置标签可编辑并同步更新对应图形标签 | 不支持 | 支持 |
| `Label List` 原地编辑 | 不支持 | 支持 |
| 修改一个标签并同步整个项目 | 不支持 | 支持 |
| 多标签映射批量转换 | 不支持 | 支持 |
| 批量操作前预览影响范围 | 不支持 | 支持 |
| 批量操作自动备份与回滚 | 不支持 | 支持 |
| 标签统计面板 | 不支持 | 支持 |
| `Insert` 快速插入边界点 | 不支持 | 支持 |
| 插点后自动选中新点 | 不支持 | 支持 |
| 图形整体移动锁定 | 不支持 | 支持 |
| 双击解锁移动 | 不支持 | 支持 |
| 长按解锁移动 | 不支持 | 支持 |
| hover 仅强调边/轻量高亮 | 不支持 | 支持 |
| 批量处理时兼容多种 JSON 编码 | 不支持 | 支持 |

上游参考：
- GitHub: https://github.com/wkentaro/labelme
- PyPI: https://pypi.org/project/labelme/

## 一、项目级标签改名

### 1. 修改一个标签，同步整个项目

现在可以做到：
- 在 `Label List` 中修改一个标签名
- 将当前项目中所有使用该标签的 polygon 一次性同步改名

触发入口：
- 双击 `Label List` 中的标签
- 在 `Polygon Labels` 中修改标签
- 使用 `Edit Label` 对话框改名

执行流程：
1. 扫描当前项目全部标注文件
2. 预览影响范围
3. 弹出确认框
4. 显示进度条
5. 批量写回所有受影响标注文件

### 2. 多标签映射批量转换

新增功能：
- `Batch Convert Labels`

快捷键：
- `Ctrl+Alt+M`

支持格式：
- `旧标签=新标签`
- `旧标签,新标签`
- `旧标签<Tab>新标签`

示例：

```text
绿油=green_oil
裂缝=crack
污渍=stain
```

适用场景：
- 项目中中文标签统一替换成英文
- 多类标签一次性标准化
- 老项目标签清洗

## 二、批量操作预览、备份与回滚

### 1. 批量操作前预览

在执行全项目改名或批量映射前，会先预览：
- 哪些旧标签会被改
- 会影响多少个 polygon
- 会影响多少个文件

作用：
- 避免误改
- 便于提前确认范围

### 2. 自动备份

每次批量改名或映射转换都会生成备份。

备份目录：

```text
.labelme_custom_history/
```

备份内容：
- 原始 JSON 文件副本
- 本次操作的 manifest 记录

### 3. 回滚最近一次批量操作

新增功能：
- `Rollback Last Bulk Operation`

快捷键：
- `Ctrl+Alt+Z`

作用：
- 恢复最近一次批量标签操作前的文件状态

适用场景：
- 映射表填错
- 改名范围超出预期
- 批量修改后想撤回

## 三、标签统计面板

新增面板：
- `Label Stats`

显示内容：
- 标签名
- polygon 数量
- 文件数量

新增功能：
- `Refresh Label Stats`

快捷键：
- `Ctrl+Alt+R`

用途：
- 统计当前项目还有哪些标签
- 检查中文标签是否已清理干净
- 查找命名不统一问题

交互：
- 双击统计表中的某个标签，可在 `Label List` 中定位该标签

## 四、标签编辑增强

### 1. `Polygon Labels` 原地编辑

位置：
- 右侧 `Polygon Labels`

行为：
- 双击后可直接编辑
- 修改后同步更新对应 polygon 的标签

### 2. `Label List` 原地编辑

位置：
- 左侧 `Label List`

行为：
- 双击标签可直接改名
- 这是项目级标签改名的主要入口

### 3. 当前图片内多选批量改名

新增功能：
- `Batch Rename Labels`

快捷键：
- `Ctrl+Alt+E`

作用：
- 对当前图片中多个选中的 polygon 一次性改成同一个标签

注意：
- 这是“当前图片内”的批量改名
- 和“整个项目同步改名”不是同一个功能

## 五、编码兼容处理

为兼容历史 Windows 中文环境生成的标注文件，批量读取 JSON 时加入了编码回退。

支持尝试的编码：
- `utf-8`
- `utf-8-sig`
- 系统默认编码
- `gb18030`
- `gbk`

作用：
- 解决“当前文件能改，其他文件因编码报错不能改”的问题

## 六、hover 显示优化

原始问题：
- 鼠标放到 polygon 上时，整块区域大面积变红，容易遮挡底图

当前优化：
- hover 时弱化大面积覆盖
- 更强调边缘反馈
- 便于看清底图纹理和边界细节

作用：
- 精修边界时更容易看图
- 减少 hover 视觉干扰

## 七、插点增强

### 1. `Insert` 快速插点

当鼠标悬停在 polygon 边附近时：
- 按 `Insert`

会执行：
- 在当前 hover 边上插入一个新点

### 2. 插点后自动选中新点

插点后会：
- 自动切换到新点
- 便于立即拖动调整位置

适用场景：
- 边界细化
- 快速补点
- 修正边缘拟合不足

## 八、图形整体移动防误触

原始问题：
- 点到 polygon 后很容易整块拖走

当前机制：
- 默认锁定整体移动
- 顶点拖动仍然正常
- 图形整体移动需要显式解锁

### 1. 显式锁定开关

新增功能：
- `Lock Shape Move`

快捷键：
- `Ctrl+Alt+L`

作用：
- 控制 polygon 整体拖动是否默认锁定

### 2. 双击解锁移动

行为：
- 双击某个 polygon
- 临时解锁该 polygon 的一次整体移动
- 拖动完成后自动重新锁定

### 3. 长按解锁移动

行为：
- 左键长按 polygon 一段时间
- 自动进入临时可移动状态
- 拖动完成后自动重新锁定

适用场景：
- 需要偶尔整体挪动 polygon
- 但日常操作以点编辑为主
- 降低误拖整块区域的概率

## 九、推荐使用流程

### 1. 统一标签命名

推荐流程：
1. 使用 `Open Dir` 打开项目目录
2. 查看 `Label Stats`
3. 若只改一个标签，用 `Label List` 直接改名
4. 若要批量统一中英文，使用 `Batch Convert Labels`
5. 执行后再用 `Refresh Label Stats` 检查是否还有旧标签残留

### 2. 精修 polygon 边界

推荐流程：
1. hover 到边附近
2. 按 `Insert` 插入新点
3. 立即拖动新点修正边界
4. 如需整体移动 polygon，再使用双击或长按解锁
